### Dagens fokus
- Vektorer
- Array references
- Strings
- Arbejde med "Imperative programming" og "Stack machine"

---

### Vektor
- Lineær sekvens af *n* dataelementer.
- Elementer tilgås via indeks (0 → n-1).
- Nogle vektorer kan ændres efter de er dannet.

---

### Tekststreng (String)
- Tekststreng = vektor af characters (tegn).
- Uforanderlige (immutable) → kan ikke ændres efter dannelse.
- Indeksering:
  - `s[0]` = første tegn
  - `len s` = længde
- Characters kan være bogstaver, tal, symboler, emojis mv.
- Eksempler:
  - `'hello'` → `s[0] = 'h'`, `s[4] = 'o'`.
  - `set s[1] = 'a'` → fejl (da strenge er immutable).

---

### Array
- Array = vektor af variabler.
- Fast længde (sættes ved oprettelse).
- Elementerne kan ændres.
- Eksempel:

```
let a = [42, true, [87, nil]]  
a[0] = 42  
a[1] = true  
a[2] = [87, nil]
```

- Kan ændres via `set a[idx] = værdi`.

### Semantik
- **Array constructor**: laver et array af længde *n*, fyldt med `nil`.
- **Array literal**: oprettes direkte med værdier.
- **Array indexing**: tilgå elementer.
- **Array length**: få længden.
- **Sideeffekter**: ændringer på array ændrer indholdet direkte (mutability).

---

### Array References
- Programmeringssproget i "Imperative programming":
- Fem slags værdier: `nil`, booleans, heltal, tekststrenge, array references.
- Værdier er immutable, når dannet.
- Arrays er *ikke værdier* → de kan ændres (mutable).
- Variabler indeholder værdier, ikke arrays → derfor bruges **referencer**.
- Array reference = abstrakt lageradresse.

---

### Hægtede strukturer
- **Linked list (hægtet liste)**: sekventiel kæde af elementer med reference til næste.
- **Doubly-linked list (dobbelt-hægtet liste)**: elementer har reference både frem og tilbage.
- Repræsenteres ofte som arrays med referencestrukturer.
- Cykliske strukturer kan give problemer i tekst-repræsentation.

---

### Tekst-repræsentation af arrays
- Arrays kan vises i tekstform (skitser).
- Cykliske strukturer → problematiske at konvertere til tekst.
- Bruges primært til visualisering af datastrukturer.

---

### Strings – formelt
- **String** = uforanderlig vektor af Unicode code points.
- **Character** = string af længde 1.
- **Glyph** = grafisk repræsentation af et tegn (afhænger af font).
- Unicode:
- `'a'` = "Latin Small Letter A" (code point 97).
- Emojis og andre symboler har også code points.

### String operationer
- **Indexing**: `s[idx]` → tegn på plads *idx*.
- **Length**: `len s` → længden.
- **Interpolation**: indsættelse af variabler i tekst.
- **Join**: sammenkædning af tekststrenge med separator.

---

### Centrale begreber
- **Immutable vs. Mutable**:
- Strings = immutable.
- Arrays = mutable.
- **References**: arrays håndteres via referencer, ikke værdier.
- **Unicode**: grundlag for tekstrepræsentation.
- **Datastrukturer**: arrays, linked lists, doubly-linked lists.
