### Modul 9 – Sikkerhedsmodenhed

#### Overordnet fokus
- Forståelse af **cybersikkerhedsmodenhed**: hvor moden er organisationens sikkerhedspraksis?
- Sammenhæng mellem **modenhed, risikostyring, governance og compliance**.
- Brug af **rammeværk** (NIST, ISO 27001, CIS18) til at vurdere og forbedre modenhedsniveau.
- Øvelse: vurdering af Hvedebros modenhed (top 10 områder, vurdering og måling).

---

### Hvad er cybersikkerhedsmodenhed?
- Udtryk for **organisationens evne til at håndtere cybersikkerhed struktureret og effektivt**.
- Viser hvor godt sikkerhedsprocesser, politikker og tekniske tiltag er implementeret og efterlevet.
- Bruges til at:
  - Identificere **styrker og svagheder**.
  - Prioritere **ressourcer og forbedringsområder**.
  - Dokumentere **compliance og risikoreduktion**.
  - Skabe **tillid** hos kunder, samarbejdspartnere og myndigheder.

---

### Hvorfor arbejde med modenhed?

| Formål                       | Effekt                                                                 |
| ---------------------------- | ---------------------------------------------------------------------- |
| **Prioritere ressourcer**    | Fokus på de vigtigste og mest risikofyldte områder.                    |
| **Øge robusthed**            | Reducerer sandsynligheden for cyberhændelser.                          |
| **Styrke compliance**        | Understøtter krav i fx GDPR, NIS2, ISO 27001 og CIS18.                 |
| **Forbedre tillid**          | Skaber troværdighed overfor kunder og samarbejdspartnere.              |
| **Definere ambitionsniveau** | Fastlægger hvor organisationen vil være sikkerhedsmæssigt (målniveau). |

---

### Typiske vurderingsområder
- Strategi og ledelsesforankring  
- Risikostyring  
- Governance og politikker  
- Awareness og kultur  
- Incident management  
- Teknisk sikkerhed (adgang, backup, netværk, mv.)  
- Leverandørstyring og outsourcing  
- Compliance og audit  
- Forretningskontinuitet og beredskab  
- Data- og personbeskyttelse  

---

### Rammeværktøjer til modenhedsanalyse

| Rammeværk                        | Fokus                                    | Bemærkninger                                                        |
| -------------------------------- | ---------------------------------------- | ------------------------------------------------------------------- |
| **NIST Cybersecurity Framework** | Risikostyring og kontinuerlig forbedring | Ofte brugt som reference for modenhedsvurdering.                    |
| **ISO 27001 / 27005**            | ISMS og risikovurdering                  | Har officielle modenhedsniveauer (licensbaseret).                   |
| **CIS18**                        | Tekniske og organisatoriske kontroller   | Kan bruges til vurdering ud fra kontrolimplementering (safeguards). |

---

### Modenhedsniveauer (eksempel)

| Niveau             | Betegnelse              | Beskrivelse                                         |
| ------------------ | ----------------------- | --------------------------------------------------- |
| **1 – Initial**    | Ad hoc                  | Ingen formaliseret proces, reaktiv tilgang.         |
| **2 – Repeatable** | Grundlæggende           | Enkelte processer gentages, men uden struktur.      |
| **3 – Defined**    | Standardiseret          | Processer er dokumenteret og delvist implementeret. |
| **4 – Managed**    | Styret og målt          | Processer overvåges, måles og justeres.             |
| **5 – Optimizing** | Kontinuerlig forbedring | Løbende læring, optimering og innovation.           |

---

### NIST-modellen og modenhed
- Bruges som reference til at vurdere modenhed på tværs af **Identify, Protect, Detect, Respond, Recover**.
- Ofte visualiseret som **radar- eller søjlediagram**, hvor hvert område scores 1–5.
- Kan sammenlignes mod **target maturity level** for at se *gaps*.

---

### Typer af modenhed
| Type                                                                                | Beskrivelse                                                                          |
| ----------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| **Politikmodenhed**                                                                 | Hvor godt politikker, standarder og procedurer lever op til rammeværkets krav.       |
| **Praksismodenhed**                                                                 | Hvor godt den faktiske operation lever op til kravene i praksis (reel efterlevelse). |
| → Kombineret vurdering giver realistisk billede af virksomhedens faktiske modenhed. |                                                                                      |

---

### Metode til modenhedsanalyse

1. **Vælg rammeværktøj** (NIST, ISO, CIS18)
2. **Definér skala og niveauer** (1–5)
3. **Kortlæg vurderingsområder og metode** (fx spørgeskemaer, interviews, evidens)
4. **Fastlæg målniveauer** (ambitionsniveau / risikoappetit)
5. **Vurder nuværende niveau**
6. **Indsaml beviser** (politikker, logs, audits, interviews)
7. **Identificér gaps** mellem nuværende og ønsket niveau
8. **Udarbejd rapport eller dashboard**
9. **Brug PDCA** (Plan–Do–Check–Act) til løbende forbedring

---

### Gratis værktøjer og ressourcer
- **Security Maturity Scan** – Nedap Security  
- **Cybersecurity Maturity Assessment (ENISA)**  
- **Sikkerhedstjekket** – Virksomhedsguiden.dk  
- **ICO Information Security Checklist** (UK)  
→ Kan bruges til en simpel baseline-analyse af modenhed.

---

### Visualisering og rapportering
- Modenhed kan præsenteres som:
  - **Dashboard** (søjle/radar-diagram)  
  - **Gap-analyse** (aktuel vs. ønsket niveau)  
  - **Heatmap** over risikoområder  
- Viser både *status* og *ambition* i ét overblik.

---

### Centrale begreber

| Begreb                               | Forklaring                                                                    |
| ------------------------------------ | ----------------------------------------------------------------------------- |
| **Modenhedsniveau (Maturity Level)** | Et tal mellem 1–5, der beskriver, hvor formaliseret og effektiv en proces er. |
| **Target Level**                     | Det ønskede fremtidige modenhedsniveau (strategisk mål).                      |
| **Gap Analysis**                     | Identificering af forskel mellem nuværende og ønsket niveau.                  |
| **Evidence**                         | Dokumentation, interviews, politikker m.m., som underbygger vurderingen.      |
| **PDCA**                             | Forbedringscyklus: Plan – Do – Check – Act.                                   |

---

### Praktisk anvendelse
- Brug modenhedsvurdering som **styringsværktøj**:
  - Skab overblik over hvor virksomheden står.
  - Prioritér sikkerhedstiltag baseret på risiko og modenhed.
  - Mål effekt over tid (fx via årlige gentagelser).
- Kombinér med **Governance og rammeværk (ISO/NIST/CIS18)** for at sikre sammenhæng.

---

### Wrap-up
- **Modenhed** = organisationens evne til at arbejde struktureret og effektivt med sikkerhed.  
- **Formålet** er at skabe overblik, prioritere indsats og støtte ledelsens beslutninger.  
- **Rammeværk** (NIST, ISO, CIS18) danner fundamentet for vurdering.  
- **PDCA-cyklussen** bruges til kontinuerlig forbedring.  
- Brug modenhedsanalyse som **kompas for udvikling, governance og compliance**.
